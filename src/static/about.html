Sobre o Projeto

Sistema de Demonstração de Algoritmos de Busca

Objetivo

Este projeto foi desenvolvido para demonstrar de forma prática as diferenças entre três algoritmos fundamentais de busca em estruturas de dados. Utilizando uma base real de 5.000 carros usados, é possível comparar a performance e entender as características de cada método.

Algoritmos Implementados

Busca Sequencial

O(n)

Percorre todos os registros linearmente. Simples de implementar, mas ineficiente para grandes volumes de dados.

Busca Indexada

O(log n)

Utiliza índice em memória para acesso rápido. Equilibra performance e uso de recursos de forma eficiente.

Busca HashMap

O(1)

Usa tabela hash para acesso instantâneo. Método mais rápido, ideal para sistemas de alta performance.

Base de Dados

O sistema utiliza um dataset real contendo 5.000 registros de carros usados, com os seguintes campos:

  • ID: Identificador único do veículo
  • Brand: Marca do carro (Toyota, Honda, Ford, etc.)
  • Model: Modelo específico (Civic, Corolla, Mustang, etc.)
  • Year: Ano de fabricação
  • Price: Preço em reais (R$)

Estrutura do Projeto

O projeto está organizado da seguinte forma:

  • db/ - Base de dados CSV e HashMap serializado
  • src/ - Código fonte Python e aplicação Flask
  • src/static/ - Páginas HTML, CSS e JavaScript
  • requirements.txt - Dependências do projeto

Como Funciona

O sistema oferece três interfaces de busca independentes, cada uma utilizando um algoritmo diferente. Ao realizar uma busca, o sistema:

  1. Recebe o termo de busca do usuário
  2. Processa a requisição no backend usando o algoritmo escolhido
  3. Retorna os resultados encontrados
  4. Exibe o tempo de execução para comparação

Complexidade Computacional

A notação Big O descreve como o tempo de execução cresce conforme o tamanho da entrada aumenta:

  • O(1): Tempo constante - não importa o tamanho da base
  • O(log n): Tempo logarítmico - cresce lentamente
  • O(n): Tempo linear - cresce proporcionalmente

Conclusão

Este projeto demonstra que a escolha do algoritmo correto pode fazer uma diferença significativa na performance de uma aplicação. Enquanto a busca sequencial é adequada para pequenos volumes, a busca HashMap se destaca em sistemas que exigem respostas rápidas e escalabilidade.
